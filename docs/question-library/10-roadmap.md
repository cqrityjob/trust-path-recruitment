# Document 10 — Question Library Roadmap

**Derives from:** all preceding Question Library documents; nests inside [Assessment DNA — Roadmap](../assessment-science/10-roadmap.md) Stage 2 (Question Library) specifically, expanding it into its own internal stages.
**Scope:** conceptual evolution stages, not a delivery schedule.

## Stage A — Current 16-Question Assessment (existing, unchanged)

Unaffected by this document set, exactly as Assessment DNA's own
Roadmap Stage 0 already established. Provides the informal precedent
(a working, if unstructured, item set) this framework formalizes going
forward — not a starting inventory to be absorbed wholesale, per the
locked architecture's convergence-path decision.

## Stage B — Question Library v1

**What it is:** the first Lifecycle-governed (Document 05) item set,
authored against this framework, covering the foundational Dimensions
(D1–D6) for the initial launch Role × Environment combination —
directly Assessment DNA Document 11 §9's recommended Phase 2B scope.
**Entry criteria:** this document set (00–10) and the Final Synthesis
approved as binding reference for content authors.
**Exit criteria:** Tier 1 (Document 06 §2) coverage sufficient to
satisfy Assessment DNA Document 06 §1's minimum-evidence floor for at
least one complete, Document 06-of-this-set-compliant Blueprint.
**Composition:** almost entirely Tier 1 (universal) content by design
— the highest-leverage, most reusable layer, built first.

## Stage C — Profession Libraries

**What it is:** Tier 2 and Tier 3 content (Document 06 §2) authored
for the specific professions and their differentiating needs
(Assessment DNA Document 07), following the shared-family/overlap
model (Document 06 §4) rather than independent per-profession
authoring.
**Entry criteria:** Stage B's Tier 1 base is stable and validated
(Document 05's Psychometric Validation reached for its core items).
**Exit criteria:** coverage-gap monitoring (Document 06 §5) shows
adequate depth for the initial set of launch professions beyond the
first Role × Environment combination.
**Sequencing note:** shared-family professions (Investigation/
Intelligence; Aviation/Maritime; Critical Infrastructure/Risk
Management) should be authored together per Document 06 §4, not in
arbitrary independent order, to realize the reuse efficiency the
shared-family model exists to provide.

## Stage D — Employer Libraries

**What it is:** the point at which individual employers' Requirement
Profiles (locked architecture, admin-created in this build) begin
drawing meaningfully on a mature enough library that their overlay
weighting (Assessment DNA Document 01 §4) has genuinely diverse Tier
1–3 content to weight, rather than a thin set with little
differentiation available.
**Entry criteria:** Stage C providing adequate Tier 2/3 depth for the
employer-relevant professions.
**Exit criteria:** demonstrated cases where distinct employers'
Requirement Profiles meaningfully differentiate using the same
underlying Blueprint content — the practical proof that the overlay
model (never modifying baseline content, per Assessment DNA Document
01 §4) works as intended at real content scale, not only in principle.

## Stage E — Adaptive Libraries

**What it is:** the item-bank depth and calibration maturity described
in Document 08 of this set, reached through continued growth of
Stages B–D's content under the same Lifecycle discipline.
**Entry criteria:** Document 08 §2–3's depth and calibration
requirements genuinely met per Dimension — not assumed once a large
raw item count exists, but verified against those specific,
per-Dimension bars.
**Exit criteria:** matches Assessment DNA Document 10's own Stage 6
exit criteria (equivalent-or-better measurement precision at reduced
respondent burden, plus a resolved adaptive-run reproducibility
model, Document 08 §5).

## Stage F — International Libraries

**What it is:** the library's expansion into markets beyond the
initial launch language(s), governed by Document 03 §8 (Cultural
Neutrality) and Document 04 §9 (Approved vs. Draft Translation status)
throughout — not a translation pass applied after the fact, but a
market-by-market re-validation following Assessment DNA Document 09
§7's cross-cultural validity requirement.
**Entry criteria:** a mature Stage B–C content base in the originating
market, with items already reviewed for cultural-neutrality
assumptions at construction time (Document 03 §8) — items *not* so
reviewed should not be fast-tracked into translation merely because
they exist.
**Exit criteria, per new market:** professional translation and
back-translation completed (not machine translation); local
subject-matter-expert review of any Domain H regulatory/procedural
content specific to that jurisdiction; re-validation of statistical
properties (Assessment DNA Document 09 §2–3) in that market before
results are treated as comparable to the originating market's — no
market is assumed to inherit another market's validation by default.
**Governing constraint that never lifts:** items in Approved-
Translation status and items in Draft-Translation status (Document 04
§9) must never be presented to a Blueprint author, or to a respondent,
as equivalent — this distinction is the single most important content-
integrity control for this stage, restated because it is the easiest
one to erode under commercial pressure to "just launch" in a new
market.

## Cross-stage principle, inherited from Assessment DNA

Identical to Assessment DNA Document 10's own closing principle,
applied at the content layer: no later stage's content decisions may
contradict an earlier stage's scientific foundation (Assessment DNA
00–09) or this framework's own principles (Documents 00–09 of this
set). Where real usage data at any stage suggests a foundational
assumption needs revision, the correct response is to revise the
foundational document through its own stated rigor — never to quietly
author around it at the content layer.
