# Document 07 — Response Format Strategy

**Derives from:** [Assessment DNA — Measurement Framework](../assessment-science/05-measurement-framework.md) (authoritative science; this document is the Question-Library-level operational companion, not a re-derivation).
**Governing principle, restated:** no universal format. Format is chosen per item against its target construct and Taxonomy category (Document 01), never by default or authoring convenience.

## 1. Formats already covered by Assessment DNA Document 05

Likert (1–5/1–7/1–10), Binary, Ranking, Forced Choice, Confidence, and
Frequency scales are specified there in full, including the reasoning
for point-count choice and the explicit rejection of 1–10 as a default
"more precise" option. This document does not repeat that reasoning —
it adds the two formats specific to Question Library operational
needs that Assessment DNA's document did not enumerate separately, and
gives a single practical cross-reference table for authors.

## 2. Situational Judgement response format

**Structurally:** a single-select choice among 3–5 realistic response
options to a scenario stem (Document 01 §2.2), keyed against expert-
derived effectiveness ranking (Document 05 of this set, Expert
Review).
**Strongest for:** the primary format for Domain B/G/D12 judgement
content, per Assessment DNA Document 04's standing in the selection-
method validity literature.
**Authoring note:** option count should stay at the lower end (3–4) for
Decision-Making items under stated time pressure (Document 01 §2.5),
where longer option sets add reading burden that itself confounds the
time-pressure construct being measured.

## 3. Ordering

**Definition:** the respondent arranges steps, actions, or events into
a correct sequence — distinct from Ranking (§4 below), which orders by
subjective priority/severity; Ordering has a single objectively
correct sequence (a procedural-fact-adjacent format).
**Strongest for:** Domain E2 (Structured Procedural Thinking), Domain
H (procedural/technical knowledge verification) — effectively a
structured variant of Knowledge Verification (Document 01 §2.9) rather
than a judgement format.
**Weak for:** any construct without one defensible correct sequence —
using Ordering for a genuinely judgement-based prioritization question
misrepresents it as having a single correct answer when it does not;
that case belongs to Ranking or SJT instead.
**Construction note:** the "correct" sequence must be verified against
actual current procedure (Document 05's Expert Review, and ongoing
Monitoring per Document 05 Stage 7 given procedural content's currency
risk, Document 01 §2.9).

## 4. Ranking (restated for contrast with Ordering)

Per Assessment DNA Document 05 §2 and Document 01 §2.12 of this set:
subjective priority/severity ordering, no single objectively correct
sequence, scored against an expert-derived reference ranking rather
than one fixed correct answer. The Ordering/Ranking distinction above
is the operational rule an author uses to choose between them —
"is there one correct sequence, or a defensible-but-contestable
priority judgement" determines the format, not item length or topic.

## 5. Multi-select

**Definition:** the respondent selects all applicable options from a
list, rather than exactly one (distinct from single-select Multiple
Choice/SJT).
**Strongest for:** Domain B3/Risk Assessment items where a realistic
situation genuinely has multiple valid concerns to identify
simultaneously (Document 01 §2.6) — forcing a single-select choice in
that case would misrepresent the construct (real risk assessment often
requires holding several concerns in mind at once, not picking the one
"most important" one). Also appropriate for certain Knowledge
Verification items where multiple facts are independently true or
false.
**Weak for:** judgement/priority constructs, where forcing a single
choice (SJT) or an explicit ranking (Ranking) is more informative than
allowing "select all that seem relevant," which can be satisfied by
low-effort over-selection.
**Scoring-adjacent caution (noted, not resolved here):** multi-select
items require a scoring approach that accounts for both correct
selections and incorrect omissions/inclusions — a Phase 2B/scoring
concern, flagged here only so authors do not assume multi-select
"just works" like single-select scoring would.

## 6. Practical cross-reference

| Format | Strongest Taxonomy category(s) | Typical Dimension target |
|---|---|---|
| Likert | Self-Reflection, Confidence-adjacent self-report | D4, D6, D11 |
| Binary | Knowledge Verification | H2 knowledge |
| Ranking | Ranking, Risk Assessment (priority form) | D7, D9 |
| Ordering | Knowledge Verification (procedural) | E2, H |
| Forced Choice | Ethical Dilemmas, targeted D3/D9 self-report | D3, D9 |
| Confidence | Confidence Rating (always paired) | D2, D9 calibration |
| Frequency | Behavioural | D3, D4, D11 |
| Situational Judgement | Situational Judgement, Decision-Making | D2, D7, D9, D12 |
| Multi-select | Risk Assessment (multi-factor form), some Knowledge Verification | D9, H |

## 7. The decision procedure, restated at the Question Library level

Identical to Assessment DNA Document 05 §4, with one addition specific
to this library: before authoring, check Document 01's category
definition for the item's chosen Taxonomy category first — the
category's own definition typically already implies the correct
format family (§6 table), and format choice should be validated
against that, not decided independently of it.
