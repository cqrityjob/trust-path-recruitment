# Document 01 — Question Taxonomy

**Derives from:** [Assessment DNA — Evidence Framework](../assessment-science/04-evidence-framework.md), [Measurement Framework](../assessment-science/05-measurement-framework.md).
**Purpose:** the content-organizing classification every Question Library entry is filed under. This is a taxonomy of item *categories*, distinct from — but mapped onto — Assessment DNA's evidence *methods* and response *formats*, which remain the authoritative science underneath it. No example questions are written here; category definitions and construction notes only.

## 1. Why a taxonomy distinct from the Evidence Framework

Assessment DNA Document 04 established which evidence-collection
*methods* are scientifically appropriate for which constructs. This
document translates that into the practical *categories* a content
author actually files an item under, browses the library by, and
authors against — closer to how a librarian's classification scheme
sits on top of, without replacing, the underlying subject taxonomy.
Several categories below map one-to-one onto an Evidence Framework
method; others (Scenario-Based, Observation, Adaptive Follow-Up)
are format/delivery categories that can carry more than one evidence
method inside them, and are named separately because they have
distinct authoring and technical requirements.

## 2. The fourteen categories

### 2.1 Behavioural
**Definition:** asks the respondent to describe or select from
described past conduct in a real or realistic situation.
**Maps to Evidence Framework:** Behavioural Questions (Past-Behavior).
**Typical target:** Domain C (Integrity), Domain E (Documentation),
Domain J (Learning Agility) — dimensions where a real track record is
a materially stronger signal than a hypothetical response (Assessment
DNA Document 03 §4).
**Construction note:** must be specific and situational, not general
("describe a specific instance," not "how often do you...") — general
framing collapses toward self-rating and loses the behavioural
method's advantage.

### 2.2 Situational Judgement (SJT)
**Definition:** presents a realistic scenario with several response
options scored against expert-derived effectiveness judgements.
**Maps to Evidence Framework:** Situational Judgement.
**Typical target:** Domain B (Judgement & Decision-Making), Domain G
(Leadership), Domain D12 (Interpersonal Influence) — the primary
method class for judgement constructs (Assessment DNA Document 04).
**Construction note:** requires genuine subject-matter-expert input to
derive a defensible key (Document 03 of this set); a scenario without
expert-validated response ranking is not yet an SJT item, only a draft.

### 2.3 Scenario-Based
**Definition:** a broader delivery category — a realistic, narrative
situation used as the stem for one or more of the other categories
below (an SJT, a knowledge check, an observation task, or a ranking
exercise can all be "scenario-based" in format).
**Maps to Evidence Framework:** format wrapper, not a method itself —
inherits whichever method the embedded question type uses.
**Typical target:** any; primarily used to increase realism/face
validity (Assessment DNA Document 09 §2) across other categories.
**Construction note:** a scenario shared across multiple embedded
questions must be reviewed for combined length/burden (Document 03 —
realism should not come at the cost of assessment length principles,
Assessment DNA Document 06 §5).

### 2.4 Ethical Dilemmas
**Definition:** presents a forced trade-off between genuinely
competing valid considerations, with no single objectively correct
resolution.
**Maps to Evidence Framework:** Dilemma Questions.
**Typical target:** Domain C (Integrity), Domain B3/D9 (Risk
Perception trade-offs) — deliberately sparing use per Assessment DNA
Document 04's caution about this method's scoring complexity.
**Construction note:** must present two (or more) options that a
reasonable, competent professional could each defend — a dilemma with
an obviously "correct" side is actually a disguised SJT and should be
filed as one.

### 2.5 Decision-Making
**Definition:** asks the respondent to make and often justify a
decision under stated constraints (time pressure, incomplete
information, competing priorities) — distinct from SJT in that the
"decision" may be open-ended within constraints rather than a fixed
option set.
**Maps to Evidence Framework:** primarily Situational Judgement,
sometimes paired with Confidence Ratings.
**Typical target:** Domain B2 (Judgement Under Incomplete
Information), Dimension D2, D8.
**Construction note:** the *constraint* (time, information
completeness) is part of the construct being measured and must be
stated precisely — a decision item without a stated constraint is
under-specified and should not leave draft status.

### 2.6 Risk Assessment
**Definition:** asks the respondent to identify, weigh, or prioritize
risks in a described situation.
**Maps to Evidence Framework:** Situational Judgement or Ranking,
depending on whether the response is a selection or an ordering.
**Typical target:** Domain B3, Dimension D9 — the calibration concern
from Assessment DNA Document 03 §2 (accurate perception, not mere
intensity of concern) governs how this category's response options
must be constructed (Document 03 of this set).
**Construction note:** response options should include plausible
over- and under-estimation choices, not only a single "correct" risk
level and obviously-wrong distractors — otherwise the item cannot
distinguish miscalibration in either direction.

### 2.7 Observation
**Definition:** presents a described or depicted scene/situation and
asks the respondent to identify what is relevant, anomalous, or
missing — a perceptual/noticing task rather than a judgement-under-
constraint task.
**Maps to Evidence Framework:** closest to a dedicated ability-style
method not previously named in Assessment DNA Document 04 as its own
category — functions similarly to a Situational Judgement item
structurally, but the construct target is detection/noticing itself
(Domain A2, Anomaly Detection), not the appropriateness of a response
to something already noticed.
**Typical target:** Domain A (Vigilance & Situational Awareness),
Dimension D1.
**Construction note:** the scene/description must contain a genuine,
expert-validated signal (something a trained observer would notice)
embedded among plausible non-signal detail — the item's validity
depends entirely on this being real, not manufactured obviousness.

### 2.8 Prior Experience
**Definition:** asks about the respondent's actual background,
training, or exposure relevant to a role or competency.
**Maps to Evidence Framework:** adjacent to Behavioural, but factual/
biographical rather than conduct-based — closer to a knowledge/
background check than a judgement probe.
**Typical target:** Domain H (Technical & Domain Knowledge), general
role-fit context rather than a specific Dimension score.
**Construction note:** must be used carefully alongside anti-
discrimination principles (Document 03 — must not function as a proxy
for a protected characteristic, e.g. certain experience patterns
correlating with age or national origin).

### 2.9 Knowledge Verification
**Definition:** asks a factual question with a determinate correct
answer (regulatory content, procedural fact, technical fact).
**Maps to Evidence Framework:** Binary or Multiple Choice with a
single correct key — the one category in this taxonomy that is
genuinely closer to ability/knowledge testing than personality/
judgement measurement.
**Typical target:** Domain H2–H3 (Regulatory & Domain-Specific
Knowledge).
**Construction note:** must be kept current against actual regulation/
procedure (Document 05's monitoring/revision stage matters especially
here, since factual content has a real expiry risk unrelated to
psychometric drift).

### 2.10 Self-Reflection
**Definition:** asks the respondent to reflect on and describe their
own past decisions, mistakes, or growth — distinct from simple
self-rating in that it asks for reasoning/narrative, not a scale
response.
**Maps to Evidence Framework:** a structured extension of self-rating,
closer in spirit to the Behavioural category but targeted at Domain
J3 (Reflective Practice) specifically.
**Typical target:** Domain J (Learning Agility), particularly J3.
**Construction note:** inherits self-report's general weakness
(Assessment DNA Document 04) for anything evaluative, but is
specifically well-suited to J3 because the *presence and quality of
reflection itself* is the signal, not the self-assessed conclusion.

### 2.11 Confidence Rating
**Definition:** captures the respondent's stated confidence in a
paired substantive response.
**Maps to Evidence Framework:** Confidence Ratings (always paired,
never standalone per Assessment DNA Document 04).
**Typical target:** Dimension D2, D9 calibration.
**Construction note:** must always be attached to a specific
substantive item with a defensible effectiveness/accuracy key —
authoring a Confidence Rating item without its pair is incomplete.

### 2.12 Ranking
**Definition:** asks the respondent to order several options by
priority, severity, or effectiveness.
**Maps to Evidence Framework:** Ranking Exercises.
**Typical target:** Domain G (Leadership/priority-setting), Domain B3/
Dimension D9 (risk-severity ordering).
**Construction note:** limit list length per Assessment DNA Document
05's working-memory-burden caution (roughly 5–7 items).

### 2.13 Forced Choice
**Definition:** requires a choice between two or more similarly
socially-desirable-sounding options, rather than independent rating of
each.
**Maps to Evidence Framework:** Forced Choice format (Assessment DNA
Document 05).
**Typical target:** Dimension D3, D9 specifically — chosen for its
bias-reduction property where straight self-rating is weakest.
**Construction note:** the paired/tetrad options must be genuinely
matched in social desirability, or the format's bias-reduction
advantage is lost — a poorly matched forced-choice pair is
functionally just an obvious single-best-answer item in disguise.

### 2.14 Adaptive Follow-Up
**Definition:** a question whose presentation depends on a prior
response within the same run — narrower than full adaptive assessment
(Document 08 of this set), this is a single, pre-authored conditional
branch (e.g. a clarifying follow-up shown only if an initial response
was ambiguous or borderline).
**Maps to Evidence Framework:** inherits whichever method the
follow-up itself uses; the novelty is the conditional trigger, not the
evidence method.
**Typical target:** any Dimension where a first-pass response leaves
genuine ambiguity worth resolving.
**Construction note:** the trigger condition and the follow-up content
must both be authored and reviewed together as one unit — an
Adaptive Follow-Up is never approved independently of its trigger
condition.

## 3. Category-to-method cross-reference

| Category | Primary Evidence Framework method | Typical response format(s) |
|---|---|---|
| Behavioural | Behavioural Questions | Frequency, Multiple Choice |
| Situational Judgement | Situational Judgement | Multiple Choice |
| Scenario-Based | (wrapper) | inherits embedded type |
| Ethical Dilemmas | Dilemma Questions | Forced Choice, Multiple Choice |
| Decision-Making | Situational Judgement | Multiple Choice, paired Confidence |
| Risk Assessment | Situational Judgement / Ranking | Multiple Choice, Ranking |
| Observation | (perceptual, SJT-adjacent) | Multiple Choice |
| Prior Experience | Behavioural (biographical) | Multiple Choice, Binary |
| Knowledge Verification | (factual/ability) | Binary, Multiple Choice |
| Self-Reflection | Self-Rating (structured) | Behavioural/narrative-adjacent |
| Confidence Rating | Confidence Ratings | Confidence scale, always paired |
| Ranking | Ranking Exercises | Ranking |
| Forced Choice | Forced Choice | Forced Choice |
| Adaptive Follow-Up | (conditional wrapper) | inherits triggered item's format |

## 4. What this taxonomy deliberately does not do

It does not write example items, does not assign categories to
specific Competencies/Dimensions beyond the "typical target" guidance
above (a Phase 2B authoring decision per item), and does not specify
scoring. Category assignment is metadata (Document 04), reviewed at
each lifecycle stage (Document 05).
